## Contexte

En accompagnant l'entreprise VTC ARRAS HDF depuis ses débuts, j'ai constaté les problèmes du terrain : gestion papier, aucun suivi automatisé, pas de visibilité financière.

Comment être pro sans outil pro ? On bricole. C'est là que j'interviens.

## Objectif

Développer une solution SaaS complète pour la gestion de l'activité d'un chauffeur VTC, couvrant tous les aspects du métier :

- Gestion des courses : création, suivi, historique
- Gestion clients : base de données, préférences, fidélisation
- Facturation automatisée : génération de factures conformes
- Statistiques et KPIs : tableau de bord avec indicateurs de performance
- Rentabilité : suivi des revenus et dépenses

## Stack Technique

- **Infrastructure** : VPS, Coolify
- **Backend/Frontend** : Next.js 14, TypeScript
- **Base de données** : PostgreSQL, Drizzle ORM
- **Authentification** : Better Auth
- **Paiement** : Stripe
- **Cache** : Redis

## Fonctionnalités clés

- Dashboard temps réel avec KPIs
- Gestion des courses (création, affectation, suivi)
- CRM clients intégré
- Facturation automatique avec génération PDF
- Statistiques avancées (CA, rentabilité, historique)
- Multi-organisation avec gestion des rôles

## Défis et Apprentissages

En solo, le code n'est que la partie émergée de l'iceberg. Entre l'infrastructure, le produit et la mise en production, c'est une expérience ultra formatrice qui m'a permis de maîtriser l'ensemble de la chaîne de développement.

## Résultat

Une application professionnelle en production, utilisée quotidiennement par VTC ARRAS HDF pour gérer son activité de manière efficace et moderne.
